# 칭찬 롤링페이퍼 웹앱 PRD

## AI에게 PRD를 줄 때의 원칙

AI가 구현을 잘 하려면 아래 형식을 지키면 됩니다.

- **한 문서에 5개 섹션을 고정 순서로** 작성 (배경 → 사용자 → 기능 → 흐름 → 참고)
- 각 기능은 **「누가 / 무엇을 / 어떤 조건에서 / 어떤 결과로」** 4요소로 씀
- **로그인 없음** 같은 제약은 `Non-Goals`와 `Constraints`에 명시
- 화면마다 **입력·출력·예외**를 표 또는 bullet로 분리
- 모호한 표현(「예쁘게」「간단히」) 대신 **측정 가능한 기준** 사용 (예: 「칭찬 1건 작성 30초 이내」)
- 마지막에 **Out of Scope**(이번에 안 만듦) 목록을 반드시 포함

---

## 1. 배경 및 목표

### 1.1 사용 목적 (Problem)

- 학급에서 긍정적同辈 칭찬(peer praise)을 구조화해, 학생들이 서로의 장점을 발견하고 표현하는 경험을 제공한다.
- 교사가 별도 계정·복잡한 설정 없이 **수업 중 5~10분 안에** 활동을 시작·마무리할 수 있어야 한다.

### 1.2 달성 목표 (Goals)

| 목표 | 성공 기준 |
|------|-----------|
| G1. 긍정적 상호작용 촉진 | 한 세션에서 학급 학생 80% 이상이 최소 1건 칭찬 작성 |
| G2. 자존감·동기 강화 | 각 학생이 **받은 칭찬 모음(롤링페이퍼)** 을 화면에서 확인 가능 |
| G3. 교사 부담 최소화 | 로그인·회원가입 없이 **학급 코드 1개**로 시작 |
| G4. 활동 결과 보존 | 세션 종료 후 **PDF 또는 이미지(PNG)** 로 개인별·학급별 저장·인쇄 |
| G5. 다양한 교실 환경 | **교사 PC/전자칠판 1대** 또는 **학생 개인 기기 다수** 모두 지원 |

### 1.3 Non-Goals (이번 버전에서 하지 않음)

- 사용자 계정, SSO, 이메일/비밀번호 로그인
- SNS 공유, 외부 URL 공개 게시
- 실시간 채팅, 댓글, 좋아요
- AI 자동 칭찬 생성
- 장기(학기) 데이터 보관·통계 대시보드

### 1.4 Constraints (제약)

- **인증 없음**: 접근은 `학급 코드`(6자리) + **학생 이름 선택**으로만 식별
- **개인정보 최소화**: 실명 대신 닉네임/번호(예: 12번 김○○) 허용, 연락처·이메일 수집 금지
- **교사 검수**: 부적절한 글 신고·삭제는 교사(호스트)만 가능
- **한국어 UI** 기본, 초등~중등 대상 톤

## 2. 사용자 분석

### 2.1 주 사용자 (Personas)

**P1. 교사 (Host)**
- 역할: 학급 세션 생성, 학생 명단 등록, 활동 진행·마무리, PDF/이미지 내보내기
- 니즈: 빠른 시작, 전체 현황 파악, 부적절 글 관리, 인쇄용 결과물

**P2. 학생 (Participant)**
- 역할: 동료에게 칭찬 글 작성, 자신의 롤링페이퍼 확인
- 니즈: 쉬운 조작, 재미있는 UI, 자신에게 온 글 확인

**P3. 학급 전체 (Display 모드 — 교사 1대 사용 시)**
- 역할: 전자칠판에 롤링페이퍼·현황 표시
- 니즈: 글자 크기 큼, 읽기 쉬운 레이아웃

### 2.2 사용 환경

| 항목 | 내용 |
|------|------|
| 장소 | 교실, PC실, 전자칠판 |
| 기기 | 교사 노트북/전자칠판, 학생 태블릿·Chromebook·스마트폰 |
| 네트워크 | 학교 Wi‑Fi (간헐적 불안정 가정 → 작성 중 자동 저장) |
| 세션 길이 | 1회 15~30분, 수업 1~2교시 내 완료 |
| 동시 접속 | 학급 25~40명 |
